# RP2040 — chip pin reference

Reference for the **Raspberry Pi RP2040** (dual-core Cortex-M0+, 133 MHz,
264 KB SRAM, no internal flash — uses external QSPI flash on dedicated
pins). Behavior described here is uniform across all RP2040-based
boards; your specific board (Pico, Pico W, third-party RP2040 module)
may break out a subset of these pins and reserve a few for onboard
hardware. Always cross-check against your board's silkscreen and
schematic.

## GPIO summary

The chip has 30 general-purpose I/O lines, GP0–GP29. There's no
catastrophic-flash-pin equivalent to the ESP32 — QSPI flash is wired
to dedicated SoC pins (SD0–SD3, SCK, /CS) that are not in the GPIO
numbering, so no GPIO toggling can corrupt the flash interface.

- **Usable for general I/O:** all 30 (GP0–GP29), subject to the
  onboard-hardware caveats below.
- **Reserved on most Raspberry Pi reference boards:** GP23, GP24,
  GP25, GP29 (see [Onboard hardware](#onboard-hardware)).
- **5 V tolerant?** No. All RP2040 GPIOs are strictly 3.3 V; driving
  a pin above 3.6 V will damage it.

## Onboard hardware

The four GPIOs the Pico reference boards wire to onboard parts. Using
these as plain GPIOs means giving up the corresponding feature — `pico
lint` warns on every literal use.

| GPIO  | Pico (non-W)                    | Pico W                                   |
|-------|---------------------------------|------------------------------------------|
| GP23  | SMPS PS pin (HIGH = PWM efficient mode, LOW = PFM power-save) | `WL_REG_ON` — cyw43 power-enable. Driving it inverts Wi-Fi state. |
| GP24  | VBUS sense (HIGH when USB is plugged in) | cyw43 SPI data (bidirectional)           |
| GP25  | Onboard LED (active HIGH)       | cyw43 SPI chip-select. Onboard LED moves to `cyw43`'s `WL_GPIO0` — use `Pin("LED", Pin.OUT)`. |
| GP29  | VSYS sense via /3 divider (read as ADC3 to estimate battery / USB voltage) | cyw43 SPI clock. Still doubles as the VSYS divider, but you can't use it without breaking Wi-Fi. |

## Analog (ADC)

12-bit ADC, 4 channels, 0 – 3.3 V range. Internal reference is the
chip's own 3.3 V rail — for absolute accuracy, anchor the rail with a
known-good regulator or measure against a precision reference.

- **ADC0** — GP26
- **ADC1** — GP27
- **ADC2** — GP28
- **ADC3** — GP29 (on Pico: VSYS / 3 divider; on Pico W: shared with cyw43 SCK)
- **ADC4** — internal temperature sensor (no GPIO)

```python
from machine import ADC
adc = ADC(26)       # GP26 → ADC0
raw = adc.read_u16() # 0..65535 (12-bit value scaled to 16-bit)
```

## DAC

The RP2040 has **no hardware DAC**. Use PWM with an external RC filter
for analog output.

## PWM

16 independent PWM slices, each with two outputs (A and B). Every GPIO
can be assigned to a slice — the SoC's IO_BANK0 mux is fully
programmable. MicroPython picks the slice for you when you call
`machine.PWM(Pin(N))`.

## PIO (Programmable I/O)

Two PIO blocks, each with 4 state machines. Lets you implement custom
bit-banged protocols (WS2812, DPI, DVI, …) without burning CPU. Each
state machine can drive any subset of GPIOs.

## UART defaults — UART(0) / UART(1)

| Peripheral | TX  | RX  | Notes                                                                 |
|------------|-----|-----|-----------------------------------------------------------------------|
| UART(0)    | 0   | 1   | Default; also reachable on GP12/GP13, GP16/GP17, GP28/GP29.           |
| UART(1)    | 4   | 5   | Also reachable on GP8/GP9, GP20/GP21, GP24/GP25.                      |

Pins are remappable; the table shows the most common dev-board
defaults. MicroPython: `UART(0, tx=Pin(0), rx=Pin(1))`.

The USB-CDC virtual serial is independent of these UARTs — it's served
by the on-die USB peripheral, not by GP0/GP1.

## I²C — convention

`machine.I2C(0, scl=Pin(1), sda=Pin(0))` and `machine.I2C(1,
scl=Pin(3), sda=Pin(2))` are the Raspberry-Pi reference defaults.
The hardware can route either channel to many other GPIO pairs (see
the RP2040 datasheet pin-function table). Pull-ups are external —
the chip has weak internal pull-ups but they're not strong enough
for I²C at typical 100 / 400 kHz speeds.

## SPI — convention

| Bus     | MISO (RX) | MOSI (TX) | SCK | CS  | Notes                                                                       |
|---------|-----------|-----------|-----|-----|-----------------------------------------------------------------------------|
| SPI(0)  | 16        | 19        | 18  | 17  | Most common default.                                                        |
| SPI(1)  | 12        | 15        | 14  | 13  | Alternate; pins remappable.                                                 |

```python
spi = machine.SPI(0, baudrate=8_000_000,
                  sck=Pin(18), mosi=Pin(19), miso=Pin(16))
```

## Power

- **3.3 V logic; no 5 V tolerance.** A 5 V level on any GPIO damages
  the SoC.
- VSYS input range: 1.8 V – 5.5 V on the Raspberry-Pi reference
  boards (the SMPS handles the conversion to 3.3 V).
- Deep-sleep current: ~ 1 mA on the Pico (the cyw43 module on Pico W
  raises this — disable Wi-Fi before sleep for low-power scenarios).

## Boot log

There's no built-in serial bootloader log on the RP2040 the way the
ESP32 has one. The BOOTSEL ROM is silent over UART; mass-storage is
its only interface. Once MicroPython is running, its REPL on USB-CDC
prints normal Python tracebacks at startup.

## Entering the BOOTSEL ROM bootloader

Three ways:

- Hold the **BOOTSEL** button while plugging in the USB cable.
- From MicroPython: `import machine; machine.bootloader()`.
- `picotool reboot -u` (works for any picoboard-library-aware firmware).

The Pico re-enumerates as a USB mass-storage device labeled
`RPI-RP2`. Drop a `.uf2` file onto it (or run `pico flash`) to
update the firmware.
