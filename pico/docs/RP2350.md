# RP2350 — chip pin reference

Reference for the **Raspberry Pi RP2350** (dual-core Cortex-M33 with
optional dual-Hazard3 RISC-V cores, 150 MHz, 520 KB SRAM, external
QSPI flash). Generation-2 successor to the RP2040. The Raspberry-Pi
reference boards using this chip are the **Pico 2** and **Pico 2 W**.

> **Untested by this tool's author.** Drafted from the RP2350 and Pico
> 2 datasheets; corrections welcome once hardware is on hand.

## GPIO summary

The RP2350 ships in two GPIO counts depending on package:

- **RP2350A** (used by Pico 2 / Pico 2 W) — 30 GPIOs, GP0–GP29.
  Matches the RP2040 footprint.
- **RP2350B** — 48 GPIOs, GP0–GP47 (larger package, not used by the
  current Raspberry-Pi reference boards).

As with the RP2040, the QSPI flash is on dedicated chip pins outside
the GPIO numbering — no GPIO toggle can corrupt it. All GPIOs are
3.3 V logic; **no 5 V tolerance**.

## Onboard hardware (Pico 2 / Pico 2 W)

| GPIO  | Pico 2                          | Pico 2 W                                  |
|-------|---------------------------------|-------------------------------------------|
| GP23  | SMPS PS pin                     | `WL_REG_ON` — cyw43 power-enable          |
| GP24  | VBUS sense                      | cyw43 SPI data                            |
| GP25  | Onboard LED                     | cyw43 SPI chip-select; LED moves to `cyw43`'s `WL_GPIO0` |
| GP29  | VSYS sense via /3 divider       | cyw43 SPI clock, also VSYS divider        |

Same wiring conventions as the RP2040-based Pico / Pico W — `pico
lint` warns on these GPIO numbers regardless of variant.

## Analog (ADC)

12-bit ADC, 4 user channels:

- **ADC0** — GP26
- **ADC1** — GP27
- **ADC2** — GP28
- **ADC3** — GP29 (VSYS / 3 on Pico 2; shared with cyw43 SCK on Pico 2 W)
- **ADC4** — internal temperature sensor

The RP2350 also adds 4-channel hysteretic comparators that don't
appear in the RP2040 — useful for ultra-low-power wake on threshold
crossings.

## Security features

The RP2350 ships with a secure boot path that the RP2040 lacks:

- ARM TrustZone for the Cortex-M33 cores.
- Hardware AES, SHA-256, and a true RNG.
- Signed UF2 support (boot rejects unsigned firmware when locked).

Default boards ship unlocked, so the existing UF2 drag-and-drop flow
still works without keys.

## PIO

Three PIO blocks (vs two on the RP2040), each with 4 state machines —
12 total. Higher PIO clocks too (up to 150 MHz).

## UART, I²C, SPI

Same peripheral count and matrix routing as the RP2040 (two UARTs,
two I²Cs, two SPIs; pins assignable to any GPIO via the IO mux).
Pico 2 reference-board defaults match the Pico's — see
[RP2040.md](RP2040.md) for the convention tables; they apply here too.

## Entering BOOTSEL

Hold the BOOTSEL button while plugging in, or from MicroPython:

```python
import machine
machine.bootloader()
```

The Pico 2 re-enumerates as a USB mass-storage device labeled
`RP2350` (note: different from the RP2040's `RPI-RP2`). Drop a
`.uf2` onto it (or run `pico flash`) to update the firmware.
